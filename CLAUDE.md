## Project Architecture

This repository is structured as a full-stack environment with a strict focus on **maximum code reuse** across our frontend platforms. 

**Directory Organization:**
- `/android`: Android application (platform-specific configurations and build files).
- `/ios`: iOS application (platform-specific configurations and build files).
- `/web`: Web frontend application (web-specific configurations and build files).
- `/backend`: Backend server, API, and database logic.

**Implementation Directive for Claude Code:**
Please implement a cross-platform framework (such as **React Native/Expo** or **Flutter**) that allows the core UI and business logic to be written once and shared across the Web, Android, and iOS platforms. Any shared code should be organized logically so it seamlessly targets the `/web`, `/android`, and `/ios` directories.

**Initial Project State & Goals:**
- The immediate goal is to establish a simple "Hello World" baseline across the three frontends (`/web`, `/android`, and `/ios`). 
- **Crucially**, this must be achieved using the cross-platform framework. Write the "Hello World" component once so that it is reused and renders correctly across all three client platforms. Ensure the backend folder has a basic server setup as well.

## Development Setup

This project uses **React Native with Expo** as the cross-platform framework, enabling maximum code reuse across Web, Android, and iOS. All shared UI components and business logic live in the `/shared` directory and automatically compile to each platform.

### Initial Setup

1. **Install dependencies across the monorepo:**
   ```bash
   npm install
   cd shared && npm install
   cd ../backend && npm install
   cd ..
   ```

2. **Or use the shortcut:**
   ```bash
   npm run install:all
   ```

3. **Environment Setup (if using backend APIs):**
   Create a `.env` file in `/backend`:
   ```
   PORT=3000
   NODE_ENV=development
   ```

## Common Commands

**Running the Apps:**
- **Web (browser):** `npm run dev:web` — Start Expo dev server for web. Opens http://localhost:19006
- **Android:** `npm run dev:android` — Start Expo dev server for Android emulator/device
- **iOS:** `npm run dev:ios` — Start Expo dev server for iOS simulator (macOS only)
- **Backend API:** `npm run dev:backend` — Start Node.js backend server on port 3000

**Building:**
- **Build all:** `npm run build` — Build optimized production bundle

**From the `/shared` directory:**
- `npm start` — Start Expo dev server (interactive menu to choose platform)
- `npm test` — Run tests

**From the `/backend` directory:**
- `npm start` — Start backend server

## Code Architecture

**Shared Frontend Code (`/shared`):**
- `App.tsx` — Main app component (entry point for all platforms)
- `components/` — Reusable React Native components shared across all platforms
- `app.json` — Expo configuration for all platform targets
- `tsconfig.json` — TypeScript configuration
- `babel.config.js` — Babel configuration for cross-platform compilation

**How It Works:**
1. Write React Native components in `/shared`
2. Expo automatically compiles and runs the **same code** on Web (React), Android (native), and iOS (native)
3. Platform-specific code can use `Platform` module: `import { Platform } from 'react-native'`
4. Styling uses React Native `StyleSheet` which works across all platforms

**Backend API (`/backend`):**
- `src/index.js` — Express server with basic API endpoints
- `GET /health` — Health check endpoint
- `GET /api/message` — Example endpoint returning a JSON message

**Platform-Specific Folders:**
- `/web/` — Web build artifacts and web-specific configs (generated by Expo)
- `/android/` — Android native code (generated via `expo prebuild` if needed)
- `/ios/` — iOS native code (generated via `expo prebuild` if needed)

## Testing

- Tests run from the `/shared` directory: `npm test`
- Uses Jest as the test runner
- Test files should follow `.test.ts` or `.test.tsx` naming convention

## Key Technologies

- **React Native** — Cross-platform UI framework (JavaScript/TypeScript)
- **Expo** — Build system and managed service for React Native (handles web, Android, iOS compilation)
- **React** — UI component library
- **TypeScript** — Type safety across the codebase
- **Express** — Backend API server
- **CORS** — Enable cross-origin requests from frontend to backend

## Styling & Patterns

- Use React Native `StyleSheet` API for all styling (works across platforms)
- All components should be functional components with hooks
- Shared components live in `components/` directory
- Use TypeScript for type safety

## Important Gotchas

- **Platform-Specific Code:** Use `Platform.OS === 'web' | 'ios' | 'android'` for platform-specific logic if needed
- **Web vs Mobile:** Expo Web runs React on the browser, while iOS/Android run true native React Native. Subtle differences may exist.
- **Backend Connection:** Frontend must be configured with backend URL (e.g., `http://localhost:3000` for development)
- **Assets:** Place static assets (images, fonts) in `/shared/assets` directory and reference them in `app.json`
- **Native Dependencies:** If adding native modules, use `expo prebuild` to generate native code; some packages may not be Expo-compatible
